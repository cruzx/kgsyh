# woodstuck

`woodstuck` 是一个面向设计评审与方案收敛的 Skill，重点帮助我们在复杂协作语境里，把一版设计、交互稿、商业化页面或概念方案收成更容易通过内部评审的版本。

它不是单纯做“审美点评”，也不是只做“产品逻辑检查”。它更像一个结合了协作上下文、业务目标、用户研究和内部过稿经验的收敛工具。

## 这个 Skill 适合干嘛

它最适合这些场景：

1. 设计稿已经有方向，但总觉得“还过不了稿”
2. 需要把散的 idea 收成更稳的方案
3. 需要在趣味性、业务表达、易用性、落地成本之间做平衡
4. 要结合历史评审语境、聊天反馈、参考基线一起判断
5. 想让输出更像“能拍板、能上线、能继续协作推进”的版本

## 它会优先帮你判断什么

这个 Skill 默认不会先讲一大段抽象道理，而是优先判断：

- 这版是不是顺当前场景
- 信息是不是更容易浏览、比较、拍板
- 业务表达有没有更清楚
- 视觉是不是耐看，而不是只刺激
- 有没有更低风险、更快上线的落地路径

如果任务明显属于酷狗内部收稿语境，它会自动切到更偏 `过稿收敛` 的判断方式：

- 先让方案 `成立`
- 再让方案 `出彩`

## 最常见的用法

### 1. 收设计稿

```text
帮我收一下这版，让它更容易过。
```

### 2. 收交互稿

```text
帮我看看这版交互为什么还不成立，按更容易拍板的方式收一下。
```

### 3. 收商业化/增长方案

```text
这个会员入口值不值得做？帮我结合当前语境和已有触点一起判断。
```

### 4. 收概念版/活动页

```text
帮我把这版概念稿收成更稳、更耐看、也更像能上线的版本。
```

### 5. 基于参考语境来判断

```text
结合已有评审语境、点击基线和研究材料，帮我判断这版该怎么改。
```


说明：

- `references/`：评审路由、点击基线、过稿镜片、轻量 spec 压缩规则
- `research/`：按主题整理的压缩研究材料
- `assets/`：辅助理解语境的图和截图
- `agents/`：配套 agent 配置

## 它和普通评审 Skill 的区别

普通评审 Skill 往往只会说：

- 哪儿不好看
- 哪儿不统一
- 哪儿还可以优化

`woodstuck` 更关注的是：

- 为什么这版还不容易过
- 卡点是在场景、信息、业务、视觉，还是落地路径
- 现在最该改的是 `P0 / P1 / P2` 哪一层
- 有没有更稳的替代实现
- 这个点位/交互/页面还能不能长成更大的产品或商业机会

也就是说，它更像一个“帮助方案通过内部评审”的协作工具，而不是一个只会做美术点评的工具。

## 自我进化机制

这份 Skill 现在带有一套显式的“双学习机制”：

- `learning/PROFILE.md`：长期稳定偏好
- `learning/EVOLUTION_RULES.md`：如何从一次对话提炼规则
- `learning/ONLINE_LEARNING_PROTOCOL.md`：什么时候该联网，联网后怎么蒸馏
- `learning/TREND_LOG.md`：外部最新信息里的高价值沉淀
- `learning/SESSION_LOG.md`：每次重要协作后的沉淀

它的目标不是假装模型会自动永久学习，而是把两类东西都留下来：

- 你这边真实协作里反复成立的判断
- 外面最新产品、交互、科技、视觉趋势里真正值得学的部分

最后把它们压缩成更强的审稿力、商业判断力和方向判断力。

## 它现在不仅能过稿，也能长判断

升级后，`woodstuck` 默认可以覆盖 5 类问题：

1. `视觉收稿`
2. `交互收稿`
3. `商业外扩`
4. `趋势吸收`
5. `创业判断`

也就是说，你以后不只可以拿它收一版稿，还可以拿它问：

- 现在外面有什么新机制值得学
- 这个需求有没有更大的商业价值
- 这个功能能不能扩成产品方向
- 如果离职创业，先做什么最值

## 安装

把整个仓库放进你的 skill 目录，并保持目录名为 `woodstuck`：

```bash
git clone <your-repo-url> woodstuck
cp -R woodstuck ~/.codex/skills/woodstuck
```

如果你使用的是其他本地 skill 根目录，也可以保持同样结构复制进去。

## Notes

- 这个 Skill 很依赖语境，所以它不是“通用美术点评器”，而是“带上下文的收敛器”。
- 如果任务不需要历史协作语境、内部评审风格或参考基线，这个 Skill 不一定是最合适的选择。
- 如果任务明确属于酷狗内部收稿语境，优先使用它会更稳。
