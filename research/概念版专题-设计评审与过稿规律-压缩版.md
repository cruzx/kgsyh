# 概念版专题｜设计评审与过稿规律压缩版

## 用途

仅在下列任务命中时读取：

- 概念版会员活动 / 运营活动 / 玩法页 / 收银台 / 免费听 / 听书 / 装扮
- 概念版 UI、海报、弹窗、卡片、首屏包装
- 需要判断“这版为什么不够容易过稿”
- 需要把概念版历史协作语境拆成可复用规则

不要泛化到普通主站产品、纯数据判断或与概念版无关的设计任务。

## 最短喂法

如果上下文很紧，只喂这一段：

- 概念版最稳定的评审信号是：`首屏更硬、按钮更明显、利益点更早露出`。
- 概念版允许趣味和世界观，但前提是 `不能牺牲结构清晰度、板式强度和质感统一`。
- 活动玩法不能只做噱头，要同时满足 `复玩动力 + 分享动力 + 奖励驱动 + 内容结合度`。
- 免费听、收银台、听书、装扮这类需求都偏好 `更强感知 + 更高转化 + 后续可复用模板化`。
- 概念版会员更适合讲成 `更纯粹 + 更好听 + 更会玩 + 更能代表你`，而不是传统会员特权堆表。

## 稳定结论

### 1. 概念版不是单纯“做得有趣”，而是“做得有趣且更容易过稿/转化”

- 这里长期关注的任务包括：
  - 会员活动
  - 玩法包装
  - 听书
  - 免费听
  - 收银台
  - 装扮
- 评审习惯不是先抠细节，而是先快速判断：
  - 方向对不对
  - 首屏够不够硬
  - 业务目标够不够清楚

### 2. 首屏和关键入口必须明显、直接

- 按钮要突出。
- 关键信息要更早露出。
- 套餐 / 歌曲 / 奖励 / 利益点不能埋得太深。
- 不能只做氛围感，不做首屏决策支持。

### 3. 活动玩法必须有持续驱动力

- 不是做一次性 gimmick。
- 长期偏好的玩法条件是：
  - 复玩动力
  - 分享/外部参与动力
  - 奖励驱动
  - 节奏推进感

### 4. 玩法要和音乐内容发生真实关系

- 活动不能只是换皮小游戏。
- 更容易被认可的是：
  - 节奏点
  - 加速感
  - 音乐内容联动
  - 玩法和听歌场景的真实关联

### 5. 概念版接受趣味，但不接受“趣味压过结构”

- 可以大胆尝试。
- 但板式、层级、质感和材质统一仍然很严。

### 6. 偏好结构化提案，不偏好只贴图

- 更容易被接受的提案结构是：
  - `需求背景`
  - `核心玩法 / 设计思考`
  - `目标 / 麻烦看看`

### 7. 偏好可以沉淀成模板的方法

- 先找试点，再沉淀成跨节日、跨场景复用模板。

### 8. 概念版会员更像“听歌体验 + 潮流功能 + 轻个性化”的组合包

- 概念版稳定成立的卖点包括：
  - 更纯粹的听歌体验
  - 更完整的音质 / 下载 / 铃声能力
  - DIY 播放器、横屏灯牌、吧唧播放器、歌词手账本等年轻化功能
  - 头像挂件、开屏、铭牌、播放器皮肤等装扮化表达

### 9. 概念版会员特权更适合“场景触发式试用”，而不是静态罗列

- 音质、下载、铃声、装扮、播放器这类特权，都更适合在具体使用场景里先感知，再转化承接。

### 10. 稀缺装扮合作更适合做“功能化 + 可展示 + 听歌主链路内可见”

- 更适合承接合作的点位包括：
  - 吧唧播放器
  - 开屏
  - 歌词手账本
  - 动态头像挂件
  - 音乐灯牌播放器
  - DIY播放器
- 这些点位共同特点是：可见度高、离听歌主链路近、适合动态和个性表达。

## 对产品 / 设计的直接含义

### 首屏表达

- 强化标题、CTA、利益点、主要对象露出。
- 避免氛围很满但主卖点很弱。

### 视觉风格

- 年轻、趣味、主题化都可以。
- 但必须保证质感统一、板式有力量、元素不拼贴、世界观自洽。

### 玩法包装

- 优先补复玩、分享、奖励和音乐联动。

### 运营沉淀

- 不只做单次活动，而要考虑后续是否能模板化复用。

### 会员特权表达

- 更适合按 `场景价值` 讲，而不是平铺后台权益。
- 更容易打动用户的顺序通常是：
  - 听歌更爽
  - 个性更强
  - 体验更潮

### 资源位与合作包装

- 概念版可以承接搜索 banner、乐库 banner、首页 banner、首页弹窗、push、歌手页 banner、活动中心、无焦点搜索、播放页挂件等资源位。
- 但更符合概念版气质的，是能引到功能试用、装扮体验和会员感知的点位。

## 典型任务怎么读

### 免费听

- 看 `用户感知` 和 `多触点露出`
- 不是只优化单个入口

### 收银台 / 生日套餐

- 看首屏、利益点、专属感、转化理由、后续复用性

### 听书 / 有声书

- 看 `基础能力完整链路 + 年轻化趣味表达`

### 活动 / 赛马 / 世界观玩法

- 看 `复玩 + 奖励 + 分享 + 音乐结合 + 成长世界观`

### 概念版会员特权页 / 合作页 / 招商页

- 看是不是把概念版会员讲成了“年轻用户愿意开的东西”。
- 优先强调纯粹听歌体验、潮流功能、稀缺装扮和场景化使用感。

## 给 `guogao-tongguan-skill` 的建议用法

命中概念版任务时，优先提取：

- `评审原则`：先看首屏、入口、利益点，再看趣味。
- `设计原则`：趣味可以放大，但结构和质感不能掉。
- `玩法原则`：复玩、分享、奖励、音乐结合。
- `运营原则`：做试点，也做模板沉淀。
- `会员表达原则`：更像体验和身份组合包，不像传统堆权益表。

## 禁止误用

- 不要把“概念版更有趣”误读成可以更乱、更花、更像海报拼贴。
- 不要把“接受创新”误读成不用讲清业务目标。
- 不要把“模板化”误读成只做套版、不做体验升级。
