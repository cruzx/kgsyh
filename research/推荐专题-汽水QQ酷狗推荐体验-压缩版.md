# 推荐专题｜Q音、酷狗、汽水推荐体验压缩版

## 用途

仅在下列任务命中时读取：

- 推荐页 / 猜你喜欢 / 个性化推荐 / 模式化听歌
- “为什么汽水更有惊喜感” 这类对比判断
- 需要判断 `保守稳定`、`热门惊喜`、`渐进探索` 哪种推荐策略更合适
- 需要给推荐产品、推荐视觉、推荐文案提供用户研究依据

不要在与推荐无关的活动视觉、收银台、弹窗、纯品牌包装任务里展开本文。

## 最短喂法

如果上下文很紧，只喂这一段：

- Q音给人的感受更像 `保守 / 熟悉 / 稳定`，适合安全感和精准熟歌，不强在惊喜。
- 汽水给人的感受更像 `热门 / 尝试 / 惊喜`，适合不知道听什么时随便听、边听边探索。
- 酷狗推荐目前给人的感受偏 `熟悉 / 稳定 / 单一重复`，核心短板是风格探索不够、惊喜不足。
- 汽水的优势不是“乱推”，而是 `先用热门建立第一印象，再做相邻风格渐进探索`。
- 用户可接受高质量/热门的二创、翻唱、AI 内容，但前提是 `质量高 + 数量受控`，一多就烦。

## 稳定结论

### 1. 三个平台在用户心智里的差异很清楚

- `Q音`：保守、熟悉、稳定，像“很像我以前的风格，但变化不大”的朋友。
- `汽水`：热门、尝试、惊喜，像“会带你试一些新东西但不会太离谱”的朋友。
- `酷狗`：熟悉、稳定、单一/重复，更像准确但不太突破的曲库型推荐。

### 2. 汽水推荐的强点不是单点热歌，而是一整套“先稳住、再探索”

- 新用户会先收到 `精准且高质量的热门歌曲`，先建立“推荐不会错”的印象。
- 进入后用 `去前奏、直接开唱`、首首衔接更强的方式提高抓耳和留存。
- 后续再用 `相邻风格渐进扩展`，而不是一下跳到很远的类型。
- 用户会把汽水理解成更适合 `不知道要听什么、想随便听听` 的场景。

### 3. 惊喜感来自三类东西

- 旋律 / 节奏 / 鼓点更抓耳。
- 怀旧经典 / 引发记忆共鸣 / 有故事感。
- 在符合自己口味的前提下，推到以前没想到但愿意接受的歌。

### 4. 推荐形式也影响感知，不只是歌本身

- 二创歌曲能带来新鲜感。
- 热评能帮助理解歌曲和情绪。
- 视频内容会增强情绪氛围和代入感。
- 但这些能力都要服从一个原则：`不要把推荐页做成内容垃圾堆`。

### 5. 用户接受 AI/翻唱/二创，但容忍度有限

- 好听就行，不天然排斥。
- AI 翻唱、二创拼接、DJ 版、改编版如果 `质量高`，可以成为惊喜来源。
- 一旦出现批量、粗糙、重复、蹭热度感强，就会迅速拉低整体体验。

## 对产品 / 设计的直接含义

### 适合保留或强化

- 给用户显式的 `模式 / 流派 / 场景标签`，帮助他们在“随便听”时快速切到所需氛围。
- 推荐策略优先采用 `相邻风格递进`，而不是一次性大跨度跳转。
- 首屏可以用 `高接受度热门歌` 建立安全感，再逐步加入边缘风格。
- 允许适量的 `二创 / 热评 / 视频氛围` 作为惊喜层，但不要成为主体。
- 如果要讲推荐卖点，优先讲：
  - `比你更懂我，但不会乱推`
  - `先推对，再推新`
  - `不知道听什么时更好用`

### 需要避免

- 同一类型 / 同一情绪 / 同一首歌的不同版本过多，造成重复疲劳。
- 为了“新鲜”而推和用户口味脱节的歌。
- 用大量 AI / 翻唱 / 二创去堆内容繁荣感。
- 把推荐差异只讲成“更多热歌”，忽略渐进探索和惊喜机制。

## 给 `guogao-tongguan-skill` 的建议用法

命中推荐任务时，只需要从本文抽下面几类信号：

- `产品原则`：先稳住口味，再做小步探索。
- `体验偏好`：惊喜可以有，但不能失控。
- `文案口径`：更懂你、但不冒犯你。
- `设计含义`：模式标签、场景切换、轻量内容辅助。

不要把本文当成 CTR 预测基线；它更适合做推荐策略和体验表达的定性护栏。

## 禁止误用

- 不要把“汽水更有惊喜”误读成“任何任务都该做得更花、更躁、更短视频化”。
- 不要把“用户接受二创/AI”误读成“可以大规模铺二创/AI 内容”。
- 不要把“Q音保守”误读成“稳定体验没有价值”；安全感和熟悉度对很多场景仍然重要。
