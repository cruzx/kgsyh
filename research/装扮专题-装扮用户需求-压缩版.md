# 装扮专题｜装扮用户需求压缩版

## 用途

仅在下列任务命中时读取：

- 皮肤 / 装扮 / 徽章 / 挂件 / 动态主题 / 专属编号
- 会员装扮权益、装扮商城、装扮激励
- 需要判断什么样的装扮更容易被接受、购买、分享
- 需要给装扮页、装扮活动、装扮商业化提供依据

不要泛化到普通会员、纯视觉海报或非个性化道具场景。

## 最短喂法

如果上下文很紧，只喂这一段：

- 装扮用户主盘并不低龄，核心集中在 `18岁以下 + 31-40岁`，且男性略多。
- 装扮价值不只是“好看”，而是 `自我代入 + 身份表达 + 社交展示 + 轻互动`。
- 用户对 `动态效果`、`专属编号`、`IP/明星联动`、`成套世界观` 的感知更强。
- 付费不是越贵越好，用户更吃 `实用耐看 + 设计感 + 对特定对象的喜爱`。
- 年轻学生价敏更强，31岁以上会更接受“免费够用、付费要真有差异”。

## 稳定结论

### 1. 用户画像不是单一二次元低龄盘

- 主力年龄覆盖 `18岁以下` 与 `31-40岁` 两头。
- 男性用户略多。
- 新一线到三线城市用户更爱使用装扮，非会员下沉更明显。
- 学生是重要主力，但并不是唯一核心人群。

### 2. 装扮消费的底层动机有两套

#### 年轻/学生向

- 更看重：
  - 自我代入
  - IP / 明星 / 社交认同
  - 屏幕美观
  - 个性展示

#### 成熟用户

- 更看重：
  - 视觉耐看
  - 不要太花
  - 使用舒服
  - 是否真有区别、值不值得

### 3. 用户对装扮“动态感”的期待比静态外观更强

- 希望有动态壁纸、互动主题、触发式小动画。
- 不只是背景动，而是希望：
  - 有场景氛围
  - 有细节反馈
  - 能随主题变化
  - 和角色/IP世界观更连贯

### 4. 专属感很关键

- 专属编号、限时限量、稀缺标识会显著提升拥有感。
- 用户会把编号、限定、特殊身份理解成可展示的资产。
- 这类元素也更容易触发主动分享到社交平台。

### 5. 用户对“印象深刻的装扮”有一套稳定偏好

- 风格质感类装扮：画面、配色、氛围舒服。
- 高辨识主题：一眼能看出世界观或对象。
- 有审美但不过花：不追求炫技堆满。
- 能和音乐、IP、角色设定形成一致感。

## 对产品 / 设计的直接含义

### 适合强化

1. `动态感`
- 动态壁纸
- 轻交互反馈
- 主题里的局部动画
- 根据主题差异设计不同动效语言

2. `专属感`
- 限定编号
- 限时限量
- 勋章式标识
- 购买后可显式展示

3. `成套世界观`
- 不要只卖单张皮肤。
- 最好是一整套：
  - 背景
  - 图标
  - 音效/交互反馈
  - 文案设定
  - 入口包装

4. `可分享性`
- 装扮页、个人页、动态页都要考虑“晒”的入口和动机。

### 需要避免

- 只做静态换皮，没有动态和反馈。
- 为了堆权益把界面做花。
- 只押热门 IP，不做基础耐看款。
- 把装扮理解成纯女性向审美生意，忽视男性和成熟用户。

## 商业化含义

- 免费皮肤已经够用，所以付费装扮必须有明显差异。
- 用户愿意为：
  - 实用耐看
  - 审美设计感
  - 特定对象的喜爱
 付费。
- 学生群体价格敏感更高；31岁以上更接受“免费够用，付费要有质差”。

## 给 `guogao-tongguan-skill` 的建议用法

命中装扮任务时，优先提取：

- `用户结构`：学生重要，但成熟用户也不能忽略。
- `设计原则`：动态、专属、成套、可晒。
- `商业化原则`：别靠纯换皮收费，要做明显差异和拥有感。
- `风险护栏`：别把装扮做得过花、一次性、无世界观。

## 禁止误用

- 不要把装扮仅理解成二次元女性用户需求。
- 不要把“动态效果”误读成越多越炫越好。
- 不要用廉价稀缺感替代真正的设计品质。
