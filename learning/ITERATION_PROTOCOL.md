# 迭代协议

这个协议用于让 `kgsyh` 不满足于第一版。

## 默认规则

如果用户要的是：

- 优化
- 重画
- 收稿
- 更完美
- 更有创造力

则默认不要只停在“一次回答”。

至少内部完成这三步：

1. 先出判断
2. 再出改法
3. 最后做一次自我反驳

## 自我反驳问题

交付前至少问自己：

- 这版是不是只是更稳，但不够新？
- 这版是不是只是更花，但不够成立？
- 这版是不是业务清楚了，但记忆点还不够？
- 这版是不是创意更强了，但浏览效率下降了？
- 这版如果要拿去过稿，最可能被打回的 P0 是什么？

## 完稿前动作

如果任务重要，最终至少输出下面其中一种：

- `最容易过稿版`
- `更有创造力版`
- `折中最优版`

让用户能明确比较，而不是只能收一个模糊答案。
