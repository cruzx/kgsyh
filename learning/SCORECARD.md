# 完稿评分卡

这个评分卡用于让 `kgsyh` 在交付前先自查，不把“差不多”当成完成。

## 使用方式

每次重要出稿、重画、收稿、改稿前，至少做一次快速评分。

默认按 10 分制粗评：

- `9-10`：可直接交付，只有轻微 polish 空间
- `7-8`：方向成立，但还有明显提升点
- `5-6`：能看，但不够稳，不建议直接交
- `0-4`：不成立，必须重构

## 五维评分

### 1. 过稿成立度

- 当前容器是否成立
- 场景是否贴合
- 是否像“能拍板”的方案

### 2. 浏览与理解效率

- 首屏是否一眼看懂
- 层级是否顺
- 是否方便比较、浏览、判断

### 3. 业务表达强度

- 利益点是否露出够早
- 业务目标是否清楚
- CTA / 入口 / 重点是否够明确

### 4. 视觉耐看度

- 是否精致而不脏
- 是否有质感而不空
- 是否能多看，不是只靠短时刺激

### 5. 创造力与记忆点

- 是否有可记住的亮点
- 是否不是模板化拼装感
- 创意是否服务页面，不是脱场景炫技

## 默认门槛

如果任务明确是“交稿”“重画”“概念版提案”“更有创造力”，则默认要求：

- 过稿成立度 `>= 8`
- 浏览与理解效率 `>= 8`
- 创造力与记忆点 `>= 7`

如果任何一项低于门槛，优先继续迭代，不要急着交。
