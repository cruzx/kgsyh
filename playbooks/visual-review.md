# Visual Review

用于 `视觉收稿`、`就是视觉`、或逻辑已过、重点转到画面收法的任务。

## 先看什么

- 整体性是否成立
- 商品感 / 容器感是否稳定
- 主次层级是否清楚
- 兴奋度是否被控制住

## 默认判断顺序

1. 场景贴合
2. 易读和易扫
3. 业务表达
4. 耐看度
5. 趣味加法

## 常见收法

- 先收成一个完整容器，再做局部气氛
- 先稳主视觉和 CTA，再补装饰
- 兴奋感优先加在局部，而不是整屏同时发力

## 常见误判

- 只追求“更炸”而丢了商品感
- 整屏都在抢主角
- 材质、发光、装饰在替代结构本身

## 输出建议

- 一句判断
- `P0`
- `保留 / 打回 / 替代收法`
