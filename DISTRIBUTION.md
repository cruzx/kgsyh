# woodstuck 可分发版说明

这份说明用于回答一个核心问题：

`woodstuck` 装到别人的电脑后，哪些能力默认就有，哪些能力需要额外配置，怎样才能继续“自主学习和进化”。

## 一句话结论

`woodstuck` 可以被分发，也可以在别人的电脑上继续成长。

但它的成长方式不是“装上就自动拥有同一个云端大脑”，而是：

- 先带走当前仓库里已经沉淀好的能力
- 再在那台电脑上继续通过 `learning/ + 自动化 + git` 演进

所以它是：

- `可复制的方法系统`
- 不是 `天然跨设备同步的共享记忆`

## 分发后默认就有的能力

只要别人安装的是当前这版 `woodstuck`，下面这些能力默认就有：

- 主 Skill 的判断框架
- `playbooks/` 里的视觉、交互、商业外扩、创业判断等打法
- `references/` 里的路由、过稿镜片、商业化设计组合体镜片
- `benchmarks/` 里的点击基线、改版复盘、数据证据
- `research/` 里的研究资料
- `assets/` 里的资料包与案例图
- `learning/PROFILE.md` 里已经稳定沉淀好的长期偏好和判断方式

也就是说，别人即使不配置自动学习，也能直接获得一套“已经被训练过”的判断方式。

## 分发后不会自动拥有的能力

下面这些能力，不会因为“装了 skill”就自动生效：

- 定时总结对话
- 定时把新经验写回 `learning/`
- 自动上传到 Git 仓库
- 跨电脑自动共享新学到的内容
- 基于最新外部信息的持续联网学习

这些能力都依赖那台电脑自己的环境配置。

## 要继续自主学习，需要哪些条件

如果希望 `woodstuck` 在别人的电脑上继续进化，至少需要这几项：

### 1. 本地可写

那台电脑需要允许 skill 修改这些目录：

- `learning/`
- `benchmarks/`
- `references/`
- `research/`

如果只能读、不能写，它就只能“使用”，不能“进化”。

### 2. 自动化能力

如果希望它按固定时间总结和沉淀，需要那台电脑也支持对应的自动任务能力，例如：

- 定时 heartbeat / automation
- 定时触发当前线程或指定线程
- 能运行写入仓库文件的流程

没有自动化时，它仍然可以学习，但只能靠“每次任务结束后手动沉淀”。

### 3. Git 权限

如果希望学习结果能被同步出去，还需要：

- 仓库 remote 可用
- GitHub SSH 或 HTTPS token 可用
- 有权限 push 到对应仓库

没有 Git 权限时，它仍然可以在本地继续长，但新经验只会停留在那台机器本地。

### 4. 联网能力

如果希望它继续吸收最新交互、科技、视觉、版式等外部变化，还需要：

- 那台电脑能联网
- 当前运行环境允许搜索和引用外部高质量来源

没有联网时，它仍然能用已有沉淀判断，但不会自动刷新趋势层。

## 三种安装级别

### A. 只安装基础版

适合：

- 只想直接使用
- 不关心后续是否自动成长

能力：

- 可以调用 `woodstuck`
- 可以使用现成的规则、资料、研究和 benchmark
- 不能自动沉淀新经验
- 不能自动同步 git

### B. 安装可学习版

适合：

- 希望越用越像本团队的判断方式
- 允许本地持续写入 `learning/`

能力：

- 拥有基础版全部能力
- 可以在任务后把有效经验写进 `learning/`
- 可以手动维护 `PROFILE / SESSION_LOG / TREND_LOG`
- 不一定自动同步 git

### C. 安装完整进化版

适合：

- 希望它长期成长
- 希望不同人协作后能把沉淀持续并回仓

能力：

- 拥有可学习版全部能力
- 可以定时总结
- 可以定时同步 git
- 可以继续联网学习并沉淀趋势

## 推荐安装顺序

如果要给别人部署，推荐按下面顺序做：

1. 安装 `woodstuck` 仓库本体
2. 确认 slash / skill 能正常调用
3. 确认本地对 `learning/` 有写权限
4. 配好 GitHub 认证
5. 再开启定时自动总结和定时同步

不要一上来先配自动化，再发现 skill 本体还没正常调用。

## 建议的团队使用方式

如果你想让这套 skill 在团队里真正越用越强，推荐这套分工：

- `主仓库`
  - 放稳定规则、结构、playbook、benchmark、研究资料
- `个人本地学习`
  - 每个人先在自己电脑上沉淀
- `定期合并回主仓库`
  - 只把已经跨任务稳定的内容推回主仓库

这样能避免两个问题：

- 每个人的短期噪音都直接污染主 skill
- 新东西虽然学到了，但永远只停留在个人电脑里

## 哪些内容适合回传主仓库

适合回传：

- 重复出现的稳定偏好
- 跨任务成立的 review method
- 已验证的商业判断规则
- 可复用的数据 benchmark
- 稳定的资料包索引和调用方式

不适合直接回传：

- 一次性情绪反馈
- 单次项目里的临时口径
- 未验证的趋势猜想
- 只在某一台电脑本地成立的实验性设置

## 如果别人只想“装了就用”

可以直接告诉对方：

- 装上后就能用
- 默认已经继承当前仓库里的能力
- 但不会自动和你共享后续新学到的东西
- 如果想让它继续成长，要额外配 `自动化 + git + 联网`

## 最小启用清单

想让分发版至少具备“可持续进化”能力，最小需要：

- `woodstuck` 仓库已安装
- skill 可正常调用
- `learning/` 可写
- git remote 可用
- GitHub 认证可用
- 能触发定时任务

## 推荐对外话术

如果你要把它发给同事或领导，可以直接这么说：

> 这不是一个只会改稿的 skill，而是一套可复制的判断系统。  
> 装上后会直接继承我们已经沉淀好的评审、商业判断和趋势吸收能力。  
> 如果把自动化和 git 同步也一起配上，它还可以在那台电脑上继续学习和进化。
